**Success Metrics & KPIs: Task Manager App**  
**Project:** TaskFlow  
**Document Owner:** Valeriia Bovsunovska, IT Project Manager  
**Last Updated:** April 2026  
**1. Purpose**  
This document defines the quantitative and qualitative metrics used to evaluate the performance, usability, and project health of the Task Manager App. 
These metrics ensure the project aligns with the goals set in the Project Charter.  
**2. Product Performance KPIs (User Experience)**  
These metrics measure how well the app serves the end user.
Metric
Target Goal
Justification
Task Creation Speed
< 3 Seconds
Supports our USP of "The 5-Second Rule" for high-speed entry.
System Latency
< 100ms
Ensures the UI feels "snappy" and responsive during navigation.
Mobile Responsiveness
100% Score
Must pass Google Lighthouse mobile-friendly audits with zero UI breaks.
Error Rate
< 1%
Percentage of user sessions that encounter a critical crash or data save error.  
**3. Project Management KPIs (Execution)**  
These metrics measure your efficiency in managing the development lifecycle.
Sprint Velocity Consistency: Maintain a variance of less than 15% between planned vs. completed tasks in each development cycle.
Documentation Coverage: 100% of core features must have corresponding User Stories and Acceptance Criteria before development begins.
On-Time Delivery: Delivery of the MVP (Minimum Viable Product) within +/- 5 days of the target roadmap date.  
**4. Quality & Growth Metrics**  
Bug Leakage: Zero "Critical" or "High" priority bugs found in the main branch after a release.
Test Coverage: Minimum 80% code coverage for backend logic (measured via GitHub Actions/Python Coverage).
User Retention (Beta): 40% of beta testers returning to the app at least 3 times in one week.  
**5. Monitoring & Tools**  
To track these metrics, we will utilize:
GitHub Insights: To track commit frequency and pull request turnaround time.
Google Lighthouse: For performance and accessibility auditing.
Vercel/Netlytics: (Or similar) for tracking load times and user session stability.
